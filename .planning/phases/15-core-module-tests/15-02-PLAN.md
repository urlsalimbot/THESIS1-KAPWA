<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test fixture → localStorage | The in-memory `localStorage` mock from `tests/setup.ts:5-12` is shared across tests in the same vitest worker — leaking token from one test to another is a real risk |
| Test fixture → global fetch | `vi.stubGlobal('fetch', vi.fn())` is per-test, but `vi.unstubAllGlobals()` in afterEach is required to prevent leakage |
| Test → production API | The new tests stub fetch with mock responses; no real `/auth/login` or `/auth/mfa/verify` calls are made during testing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-02-01 | Tampering | `auth-context.login.test.tsx` login success test | mitigate | The test asserts the user object + token via deep equality; the mock fetch response is a hard-coded object literal (not from the API surface); no production code path can mutate the fixture |
| T-15-02-02 | Information Disclosure | localStorage token leak across tests | mitigate | `localStorage.clear()` in `beforeEach` per D-13 file split pattern; `vi.unstubAllGlobals()` in `afterEach` (mirrors the existing `auth-context.test.tsx:19-22` pattern) |
| T-15-02-03 | Spoofing | Test could pass with a hard-coded response from a malicious server | accept | The mock response is a hard-coded literal in the test file; no network is involved; the test asserts the contract, not the security posture of the server |
| T-15-02-04 | Repudiation | Test could pass when the wrong code path is hit (e.g., login instead of resolveMfa) | mitigate | Each test mocks a unique fetch URL sequence; tests assert on the function's return value (login returns MFA object, resolveMfa clears mfaChallenge) — the assertions are specific to the code path being tested |
| T-15-02-05 | Elevation of Privilege | cancelMfa test could mask an accidental fetch | mitigate | The test asserts `fetchMock.mock.calls.length === 1` after cancelMfa — any new fetch call would increase the count and fail the test |
| T-15-02-SC | Tampering | npm package install | accept | No new package installed in this plan; only test files added |
</threat_model>
