  <action>
    Read the current `kapwa-client/src/index.css` (which now includes Plan 1's expanded `:root` and `.dark` block inside `@layer base`).

    Apply these structural changes (in order):

    **1. Add layer order declaration at the TOP of the file, BEFORE `@tailwind base`:**
    ```
    @layer base, components, legacy, utilities;
    ```
    This establishes the cascade layer order: base < components < legacy < utilities. The last layer (utilities) has highest priority — Tailwind/shadcn utility classes always beat legacy class selectors.

    **2. Remove the entire old components-layer block** (currently lines ~49-146 containing `.page-header`, `.page-title`, `.form-input`, `.table`, `.badge-*`, `.stat-*`, `.login-*`, etc.). This is the block currently wrapped in the old layer directive.

    <!-- planner-discipline-allow: @layer components -->

    **3. Insert a new `@layer legacy` block AFTER `@tailwind utilities;`** (around line 3):
    ```
    @layer legacy {
      /* Legacy component classes (kept for backward-compatibility until Phase 11 migration) */
      .page-header { @apply mb-4; }
      ...
    }
    ```
    The content of `@layer legacy` is IDENTICAL to what was inside the old components-layer block — no CSS rules are changed (per D-06). Only the `@layer` directive name changes.

    **4. Position:** The `@layer legacy` block is physically placed between `@tailwind utilities;` and the `@layer base` block (or between `@tailwind utilities;` and `@import url(...)`).

    **5. The `@layer base { ... }` block (with `:root`, `.dark`, `*` reset, `body`, `h1-h6` styles) remains unchanged.**

    The final file structure should be:
    ```
    @layer base, components, legacy, utilities;

    @tailwind base;
    @tailwind components;
    @tailwind utilities;

    @import url('https://fonts.googleapis.com/...');

    @layer legacy {
      /* All content from the old components-layer block, unchanged */
    }

    @layer base {
      :root { /* expanded with Plan 1 tokens */ }
      .dark { /* Plan 1 dark mode colors */ }
      * { @apply border-border; ... }
      body { ... }
      h1, h2, h3, h4, h5, h6 { ... }
    }

    @media print { ... }
    ```

    **Important**: Do NOT change any CSS rule body. Only change:
    - Add `@layer base, components, legacy, utilities;` at line 1
    - Rename the old layer directive from components to legacy
    - Move the legacy block to after `@tailwind utilities`
  </action>
