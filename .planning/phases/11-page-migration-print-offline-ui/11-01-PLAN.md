    ```

    ```

    - Accept `cachedAt` as a Unix timestamp (milliseconds since epoch), or `undefined`
    - If `cachedAt` is undefined: return `{ isStale: false, ageDisplay: '' }`
    - Compute `elapsed = Date.now() - cachedAt`
    - Threshold: `STALE_AFTER_MS = 5 * 60 * 1000` (5 minutes, per D-17)
    - `isStale = elapsed >= STALE_AFTER_MS`
    - Format `ageDisplay`:
      - less than 1 min: `less than 1 min`
      - less than 60 min: `X min` (Math.floor)
      - 60 min or more: `X hr Y min` (Math.floor divisions)
      - 24 hr or more: `X days Y hr`
    - Use `useEffect` with a 30-second interval timer that re-computes the age display (so the "X min ago" text updates in real-time without page refresh). Clean up the interval on unmount.
    - Export `STALE_AFTER_MS` constant so consumers can reference the threshold.
    - File location: `kapwa-client/src/hooks/use-cache-staleness.ts`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → SyncQueuePanel | Queue data read from localStorage — no network boundary, but stale/conflicting queue state could cause incorrect display |
| Print stylesheet | CSS-only — no execution boundary, but CSS `@media print` `content:` properties on `a[href^="http"]::after` could leak internal URLs to print output |
| ConflictResolutionDialog UI | Display-only — no writes to server; resolve actions update localStorage only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | `a[href^="http"]::after` in print CSS | accept | Internal URLs in print output are acceptable for audit purposes; no secrets in URLs per coding convention |
| T-11-02 | Tampering | SyncQueuePanel queue operations | mitigate | Queue updates only modify localStorage state; `updateQueueItem` and `removeQueueItem` operate on local data only — no server writes from UI components |
| T-11-03 | Denial of Service | SyncQueuePanel 2s polling | accept | Polling only active when panel is open; queue size bounded by localStorage limits (~5MB); items removed after sync |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages required — all shadcn components (Sheet, Dialog, Badge, Button, ScrollArea) already installed from prior phases per RESEARCH.md Package Legitimacy Audit |
</threat_model>
