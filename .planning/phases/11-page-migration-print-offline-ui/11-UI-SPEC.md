## Spacing Scale

Declared values (multiples of 4px, mapped to CSS custom properties):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, filter pills |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, PageShell gap |
| xl | 32px | Layout gaps, sidebar width |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing, print margins |

**Exceptions:**

| Context | Value | Reason |
|---------|-------|--------|
| Print document margins | **20mm** (~76px) | A4 government report spec (D-07) — uses CSS `@page` margin, not Tailwind scale |
| Print base body font | **12pt** | Government document standard (D-07) — pt units outside rem/px scale |
| Current print stylesheet | **15mm** | Existing `@media print` `@page { margin: 15mm }` — **must be updated to 20mm** per D-07 |
| Sync queue items | **12px** gap | Compact list density in Sheet panel — tighter than default md(16px) |
| 44px touch targets | **min-height: 44px** | Already enforced globally via `@layer components` in index.css (Phase 10) |

## Page Migration Contract

### Page Classification

| Priority | Page Group | Pages | Key Patterns |
|----------|-----------|-------|-------------|
| P0 — Pattern Setter | Dashboard | DashboardPage, ClaimantDashboardPage | PageShell + CardGridSkeleton + stat cards + KPI grid |
| P1 — Data Tables | Beneficiaries, Cases | BeneficiariesPage, CasesPage, InterventionsPage | PageShell + DataTable + TableSkeleton + EmptyState-variant |
| P1 — Detail Views | BeneficiaryView, CaseTracker, ApprovalPipeline | BeneficiaryViewPage, CaseTrackerPage, ApprovalPipelinePage | PageShell + CardGridSkeleton + FormSkeleton + status badges |
| P2 — Print Documents | CSR, Access Card | CsrPage, AccessCardPage, AccessCardPrintView | PageShell + DataTable + Print CSS + signature blocks |
| P2 — Form Heavies | Intake, IRF | IntakePage, IrfPage, IrfDetailPage | PageShell + FormSkeleton + shadcn Form + EmptyState |
| P3 — Admin & Utilities | Admin, MfaSetup, Filing, Messages | AdminPage, MfaSetupPage, FilingPage, MessagesPage | PageShell + DataTable + TableSkeleton |

### Migration Sequence (from D-02)

1. **Dashboard** — sets pattern for all other pages
2. **Beneficiaries** (list + view) — DataTable pattern
3. **Cases & Interventions** — DataTable + detail patterns
4. **Access Card & CSR** — DataTable + print patterns
5. **IRF** — form-heavy pattern
6. **Admin, MFA, Filing, Messages** — utility patterns
7. **Claimant Dashboard** — alternate dashboard pattern
8. **CaseTracker & ApprovalPipeline** — complex-specialized pages

### Required States Per Page

Every migrated page MUST implement these states via Phase 10 shared components:

| State | Component | Condition |
|-------|-----------|-----------|
| Loading | `TableSkeleton` / `CardGridSkeleton` / `FormSkeleton` | Initial data fetch (cold load) |
| Empty | `EmptyState variant="no-data"` | Zero records in list/dashboard, route to create |
| No Results | `EmptyState variant="no-results"` | Search/filter returns zero matches |
| Offline | `EmptyState variant="offline"` | `navigator.onLine === false` (or fetch error) |
| No Access | `EmptyState variant="no-access"` | Insufficient role/permission |
| Cached Data | Cache staleness badge in PageShell `description` prop | Data age > 5 min (D-17) |
| Error | Global `ErrorBoundary` in Layout.tsx catches unhandled render errors | Any uncaught exception |

### Print Styles Per Page

| Document | Layout | Elements | Print Trigger |
|----------|--------|----------|--------------|
| CSR | A4 portrait, 20mm margins, header/footer, signature block last page | MSWDO header, case info, assessment sections, signature | Button in CsrPage detail view |
| Access Card | A4 portrait, card layout | Beneficiary info, QR code, issue/expiry date, seal | Button in AccessCardPage / AccessCardPrintView |
| Intervention Log | A4 portrait, column-adjusted table | Service entries table, case info, worker/clinic signatures | Button in InterventionsPage detail |
| IRF | A4 portrait, form layout | Incident details, parties involved, encrypted references, signature | Button in IrfPage / IrfDetailPage |

### PageShell Cache Integration

The existing PageShell component receives a new optional `cachedAt?: number` prop. When provided and age > 5 min, appends cache staleness text to `description`:

```
"Case records" → "Case records — Cached data — last sync 12 min ago"
```

## Interaction Design

### Sync Queue Panel

| Interaction | Behavior |
|-------------|----------|
| Open panel | Click sync status button in sidebar/topbar → Sheet slides from right |
| Close panel | Click X button, click overlay, press Escape, or navigate |
| Real-time updates | Items transition through states: pending → syncing (spinner) → done (checkmark) / failed (X) |
| Retry failed | Click "Retry" → re-queues item for sync → status becomes "pending" |
| View conflict diff | Click "View Diff" → opens ConflictResolutionDialog showing local vs server values side by side |
| Resolve conflict | Choose Keep Local / Keep Server / Keep Both → sync engine applies resolution → item removed from queue |
| Remove item | Swipe left (mobile) or click menu → Remove → confirmation dialog → item removed |
| Empty state | No items → `CheckCircle` icon + "All caught up" heading |

### Cache Staleness

| Condition | Behavior |
|-----------|----------|
| Data fetched, lastSync < 5 min ago | No badge — data considered fresh |
| Data fetched, lastSync >= 5 min ago | "Cached data — last sync X min ago" in PageShell description |
| No lastSync timestamp | "Cached data — not yet synced" fallback |
| User triggers refresh | Data re-fetched, lastSync updated, badge disappears (if within 5 min) |

### Print

| Interaction | Behavior |
|-------------|----------|
| Trigger print | Button labeled "Print Document" → `window.print()` called |
| @media print hides | Sidebar, Topbar, BottomNav, action buttons, breadcrumbs — all `.no-print` |
| Print layout | A4 portrait, 20mm margins, serif body, Lexend headings |
| Page headers/footers | MSWDO header + logo on first page, page X of Y + print date on every page |
| Signature blocks | Last page only, `break-inside-avoid` to prevent split across pages |
