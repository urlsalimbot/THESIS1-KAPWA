    Add a new `@layer components` block to `src/index.css` that enforces 44px minimum touch targets on all interactive elements, per D-27 (global CSS override approach).

    Insert the block AFTER the `@layer legacy` block closes (line 107) and BEFORE the `@layer base` block (line 109). The new block should be placed as a separate `@layer components` rule.

    ```css
    @layer components {
      /* 44x44px touch targets for mobile accessibility (Phase 10) */
      button, a[href], input, select, textarea, [role="button"] {
        min-height: 44px;
        min-width: 44px;
      }
      /* Opt-out class for small decorative/inline elements */
      .touch-sm {
        min-height: unset;
        min-width: unset;
      }
    }
    ```

    Per D-25: applies to BOTH existing interactive elements AND all new components (the global rule handles this automatically).
    Per D-26: primary targets are inline action buttons, icon-only buttons, clickable badges/pills — all covered by the `button, a[href], input, select, textarea, [role="button"]` selector.
    The `.touch-sm` opt-out allows individual small decorative elements to revert (e.g., a 20px indicator dot, an inline decorative badge). Per Pitfall 5 from RESEARCH.md, this prevents the global override from breaking shadcn components that are intentionally small.

    The rule should be placed in `@layer components` (not `@layer base` or `@layer legacy`) to ensure it has appropriate specificity without overriding utility classes. Since it targets raw elements, it won't conflict with `@layer base`'s existing element resets.

    No UI-visible change should occur on desktop — shadcn Button is `h-10` and this adds `min-h-11` (44px = 11 in Tailwind spacing). The net effect on desktop is minimal; the main impact is on mobile where small inline buttons (< 44px) are enlarged.
  </action>
