## Visual Hierarchy & Focal Points

| Priority | Element | Rationale |
|----------|---------|-----------|
| Primary | Sidebar navigation | User's primary action is route selection. Active nav item highlighted with `bg-muted text-foreground`. This is where users spend most of their interaction time. |
| Secondary | Main content area (`<main>`) | Where selected route content renders. Breadcrumb trail provides navigation context. |
| Tertiary | Topbar | Utility actions (notifications, messages, user menu, dark mode toggle, search). Support role Γאפ peripheral to primary task. |

**Focal point enforcement:** No competing visual elements draw attention away from the sidebar-content axis. The sidebar sticks permanently on desktop (256px, `sticky`), the main content fills remaining space. Accent color is reserved exclusively for interaction feedback (focus rings, hover states) Γאפ never for chrome or decoration.

## Typography

Font families (inherited from Phase 7 design tokens):
- **Body:** Source Sans 3 (`--font-body`)
- **Heading:** Lexend (`--font-heading`)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (normal) | 1.5 | Nav item labels, breadcrumb text, user menu items, placeholder text |
| Label | 12px | 600 (semibold) | 1.0 | Nav group headers (uppercase, `tracking-wider`), role badge text |
| Heading | 14px | 600 (semibold) | 1.25 | "KAPWA" logo brand text (uses Lexend font) |
| Display | 16px | 600 (semibold) | 1.5 | User full name in DropdownMenu profile card |

**Weight summary:** Exactly 2 weights used Γאפ 400 (normal) for body/display text and 600 (semibold) for all headings, labels, and emphasis. No 500 (medium) or 700 (bold) weights used. This ensures a clean, readable hierarchy without unnecessary weight variation.

**Note on "Skip to content":** Uses `sr-only` / `focus:not-sr-only` pattern Γאפ font size inherits from body (14px), weight 600 (semibold) for visibility on focus.

**Active nav highlight:** `bg-muted text-foreground` with `hover:bg-muted/50 hover:text-foreground` for inactive items. No accent color used for navigation Γאפ ensures the accent remains distinctive for interaction feedback.

## Interaction States

### Sidebar Navigation

| State | Visual |
|-------|--------|
| Default (inactive) | `text-muted-foreground hover:bg-muted/50 hover:text-foreground` |
| Active (current route) | `bg-muted text-foreground` Γאפ background fills full item width |
| Route matching | Exact match OR path prefix (non-`/` routes) `location.pathname.startsWith(item.path)` |
| Group hidden | Entire group removed from DOM when `filter()` returns 0 visible items for user's role |

### Desktop Sidebar (>=1024px)

- Always visible, `w-64` (256px), sticky below topbar
- `sticky top-16` Γאפ pinned below the fixed topbar
- `h-[calc(100vh-4rem)]` Γאפ fills remaining viewport
- Independent scroll via `<ScrollArea>`

### Mobile Sidebar (<1024px)

- Hidden by default (CSS: `lg:block hidden`)
- Opens via hamburger button (topbar left) Γאפ `<Menu>` icon
- Rendered as shadcn `<Sheet>` with `side="left"`
- Same nav content as desktop via shared `SidebarNavContent` render function
- Overlay: `bg-black/80` (default Sheet)
- Close on nav item click, close on overlay click, close on Escape key (Radix built-in)

### Topbar

| Element | Interaction |
|---------|-------------|
| Logo + title | Links to `/` Γאפ KAPWA brand + Norzagaray MSWDO subtitle |
| Role badge | Static display Γאפ `Badge variant="secondary"`, hidden on mobile, inline on md+ |
| Search input | Visual-only placeholder (`rounded-full bg-muted/50`). No search wiring Γאפ deferred. |
| Notifications bell | Existing `NotificationsDropdown.tsx` Γאפ preserved, unchanged |
| Messages envelope | Existing `MessagesPopover.tsx` Γאפ preserved, unchanged |
| Help circle | Static icon Γאפ no content. Deferred. |
| Avatar | Triggers `<DropdownMenu>`. Shows initials (2-letter abbreviation of full name) via `AvatarFallback`. |
| Hamburger (mobile) | Toggles Sheet sidebar open/close. Only visible `<lg`. |

### User DropdownMenu

| Item | Behavior |
|------|----------|
| Profile card (top) | Display name + role. No action Γאפ visual header. |
| Dark Mode toggle | `onClick={setTheme(theme === 'dark' ? 'light' : 'dark')}`. Sun/Moon icon. Mount-guarded: renders placeholder `<div className="w-9 h-9" />` until `mounted` is true (hydration guard for next-themes `resolvedTheme`). |
| Settings Γזע Change Phone | `DropdownMenuItem asChild` Γזע `<Link to="/settings">`. Route not yet implemented (Phase 12). |
| Settings Γזע Change Password | `DropdownMenuItem asChild` Γזע `<Link to="/settings">`. Same placeholder route. |
| `Separator` | Visual divider |
| Logout | `DropdownMenuItem` with logout icon. Calls auth-context logout function. Standard navigation action. |

### Breadcrumbs

- Derived from `location.pathname` via `createBreadcrumbs()` resolver function
- Each segment mapped through `BREADCRUMB_LABELS` constant (longest-prefix match)
- First crumb: always "Dashboard" linking to `/`
- Intermediate crumbs: `<BreadcrumbLink asChild>` Γזע `<Link>` Γאפ clickable
- Last crumb: `<BreadcrumbPage>` Γאפ plain text, `aria-current="page"`
- Parameterized routes: generic label such as "View Beneficiary", "View IRF", etc. (entity name resolution deferred)
- Keys: use `crumb.href` to prevent React key warnings on parameterized route changes

### Skip-to-Content

- First focusable element in Layout.tsx (before Topbar)
- `href="#main-content"` targeting `<main id="main-content">`
- Hidden by default: `sr-only`
- Visible on keyboard focus: `focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-[100] focus:px-4 focus:py-2 focus:bg-accent focus:text-accent-foreground focus:rounded-md`
- Tab order: 1. Skip to content Γזע 2. Topbar Γזע 3. Sidebar Γזע 4. Main content

### Dark Mode Toggle (Accessibility)

- Must render a placeholder of identical dimensions while `mounted` is false to prevent layout shift:
  ```tsx
  const [mounted, setMounted] = useState(false);
  useEffect(() => setMounted(true), []);
  if (!mounted) return <div className="w-9 h-9" />;
  ```
- Toggle icon: Sun when in dark mode (switch to light), Moon when in light mode (switch to dark)
- Uses `resolvedTheme` (not `theme`) to correctly reflect actual applied theme
