## Color

All values are CSS custom properties from Phase 7 design tokens. Light / Dark pairs shown.

| Role | Light Value | Dark Value | Usage in This Phase |
|------|-------------|------------|---------------------|
| Dominant (60%) | `--background` #F8FAFC | #0F172A | Main content area (`<main>`, `<body>`) |
| Secondary (30%) | `--card` #FFFFFF | #1E293B | Sidebar background, topbar background, dropdown menu, sheet content |
| Muted surface | `--muted` #E8ECF1 | #334155 | Active nav item background (`bg-muted`), hover states (`hover:bg-muted/50`), search input background |
| Accent (10%) | `--accent` #0369A1 | #38BDF8 | **See reserved-for list below** |
| Destructive | `--destructive` #DC2626 | #EF4444 | Not used in this phase (no destructive actions) |
| Border | `--border` #E2E8F0 | #334155 | Sidebar right border, topbar bottom border, separators, sheet borders |
| Foreground | `--foreground` #020617 | #F8FAFC | Primary text color |

| Primary | `--primary` #0F172A | #F8FAFC | KAPWA logo brand text, primary button backgrounds |

**Accent reserved for:**
- Focus rings on all interactive elements (keyboard focus visible ring)
- Search input focus border state (`focus:border-accent`)
- Skip-to-content link background on focus
- Any interactive element hover underline (non-critical, for emphasis)
- **NOT** used for nav item active state (that uses `bg-muted text-foreground`) вҖ” this is an intentional choice to reserve accent for interaction feedback, not navigation chrome.

**Active nav highlight:** `bg-muted text-foreground` with `hover:bg-muted/50 hover:text-foreground` for inactive items. No accent color used for navigation вҖ” ensures the accent remains distinctive for interaction feedback.

## Interaction States

### Sidebar Navigation

| State | Visual |
|-------|--------|
| Default (inactive) | `text-muted-foreground hover:bg-muted/50 hover:text-foreground` |
| Active (current route) | `bg-muted text-foreground` вҖ” background fills full item width |
| Route matching | Exact match OR path prefix (non-`/` routes) `location.pathname.startsWith(item.path)` |
| Group hidden | Entire group removed from DOM when `filter()` returns 0 visible items for user's role |

### Desktop Sidebar (>=1024px)

- Always visible, `w-64` (256px), sticky below topbar
- `sticky top-16` вҖ” pinned below the fixed topbar
- `h-[calc(100vh-4rem)]` вҖ” fills remaining viewport
- Independent scroll via `<ScrollArea>`

### Mobile Sidebar (<1024px)

- Hidden by default (CSS: `lg:block hidden`)
- Opens via hamburger button (topbar left) вҖ” `<Menu>` icon
- Rendered as shadcn `<Sheet>` with `side="left"`
- Same nav content as desktop via shared `SidebarNavContent` render function
- Overlay: `bg-black/80` (default Sheet)
- Close on nav item click, close on overlay click, close on Escape key (Radix built-in)

### Topbar

| Element | Interaction |
|---------|-------------|
| Logo + title | Links to `/` вҖ” KAPWA brand + Norzagaray MSWDO subtitle |
| Role badge | Static display вҖ” `Badge variant="secondary"`, hidden on mobile, inline on md+ |
| Search input | Visual-only placeholder (`rounded-full bg-muted/50`). No search wiring вҖ” deferred. |
| Notifications bell | Existing `NotificationsDropdown.tsx` вҖ” preserved, unchanged |
| Messages envelope | Existing `MessagesPopover.tsx` вҖ” preserved, unchanged |
| Help circle | Static icon вҖ” no content. Deferred. |
| Avatar | Triggers `<DropdownMenu>`. Shows initials (2-letter abbreviation of full name) via `AvatarFallback`. |
| Hamburger (mobile) | Toggles Sheet sidebar open/close. Only visible `<lg`. |

### User DropdownMenu

| Item | Behavior |
|------|----------|
| Profile card (top) | Display name + role. No action вҖ” visual header. |
| Dark Mode toggle | `onClick={setTheme(theme === 'dark' ? 'light' : 'dark')}`. Sun/Moon icon. Mount-guarded: renders placeholder `<div className="w-9 h-9" />` until `mounted` is true (hydration guard for next-themes `resolvedTheme`). |
| Settings вҶ’ Change Phone | `DropdownMenuItem asChild` вҶ’ `<Link to="/settings">`. Route not yet implemented (Phase 12). |
| Settings вҶ’ Change Password | `DropdownMenuItem asChild` вҶ’ `<Link to="/settings">`. Same placeholder route. |
| `Separator` | Visual divider |
| Logout | `DropdownMenuItem` with logout icon. Calls auth-context logout function. Standard navigation action. |

### Breadcrumbs

- Derived from `location.pathname` via `createBreadcrumbs()` resolver function
- Each segment mapped through `BREADCRUMB_LABELS` constant (longest-prefix match)
- First crumb: always "Dashboard" linking to `/`
- Intermediate crumbs: `<BreadcrumbLink asChild>` вҶ’ `<Link>` вҖ” clickable
- Last crumb: `<BreadcrumbPage>` вҖ” plain text, `aria-current="page"`
- Parameterized routes: generic label such as "View Beneficiary", "View IRF", etc. (entity name resolution deferred)
- Keys: use `crumb.href` to prevent React key warnings on parameterized route changes

### Skip-to-Content

- First focusable element in Layout.tsx (before Topbar)
- `href="#main-content"` targeting `<main id="main-content">`
- Hidden by default: `sr-only`
- Visible on keyboard focus: `focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-[100] focus:px-4 focus:py-2 focus:bg-accent focus:text-accent-foreground focus:rounded-md`
- Tab order: 1. Skip to content вҶ’ 2. Topbar вҶ’ 3. Sidebar вҶ’ 4. Main content

### Dark Mode Toggle (Accessibility)

- Must render a placeholder of identical dimensions while `mounted` is false to prevent layout shift:
  ```tsx
  const [mounted, setMounted] = useState(false);
  useEffect(() => setMounted(true), []);
  if (!mounted) return <div className="w-9 h-9" />;
  ```
- Toggle icon: Sun when in dark mode (switch to light), Moon when in light mode (switch to dark)
- Uses `resolvedTheme` (not `theme`) to correctly reflect actual applied theme

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Skip-to-content (LAY-04) | First focusable element. `<a href="#main-content">` вҶ’ `<main id="main-content">`. Visible only on keyboard focus via `sr-only focus:not-sr-only`. |
| ARIA landmarks | `<header>` for topbar, `<aside>` for sidebar, `<main>` for content, `<nav>` for sidebar navigation. |
| Breadcrumb ARIA | `<nav aria-label="Breadcrumb">` вҖ” from shadcn Breadcrumb component. `aria-current="page"` on last crumb via `<BreadcrumbPage>`. |
| DropdownMenu ARIA | Radix DropdownMenu provides `role="menu"`, `aria-orientation`, arrow key navigation, focus management. |
| Sheet ARIA | Radix Dialog provides `role="dialog"`, `aria-modal="true"`, focus trapping, Escape key dismissal. |
| Dark mode toggle | No ARIA needed вҖ” `<button>` with icon conveys state visually. Label "Dark Mode" / "Light Mode" text alongside icon. |
| Active nav item | Visual highlight only (`bg-muted text-foreground`). No ARIA `aria-current="page"` вҖ” breadcrumb already provides this. |
| Focus order | Skip-link вҶ’ Topbar (hamburger вҶ’ logo вҶ’ search вҶ’ notifications вҶ’ messages вҶ’ help вҶ’ avatar) вҶ’ Sidebar nav items вҶ’ Main content. |
| Color contrast | All text colors use system tokens verified in Phase 7. `bg-muted text-foreground` on active nav meets 4.5:1+ ratio. |
