For each of these 7 page test files, add an axe a11y assertion. Process one page at a time: add the assertion → run the test → fix violations → commit.

**Important difference from Plan 17-02:** These pages mostly use simple `render()` with `BrowserRouter` or `MemoryRouter` — they do NOT use `renderWithSWR` (no data fetching via SWR for most). Adapt the render wrapper to match each page's existing pattern per D-08.

**Per-page render patterns:**

1. **AuditorPage** — Read file first. Likely uses SWR (data-display page). If so, reuse any existing helper or inline `render(<MemoryRouter><AuditorPage /></MemoryRouter>)` with mock data.

2. **BeneficiaryViewPage** — Read file first. May use route params. Use inline `render()` matching existing pattern.

3. **ClaimantDashboardPage** — Read file first. Likely uses SWR for claimant data. Reuse existing helper.

4. **ContactPage** — Read file first. May be public page with simple render or SWR for contact data.

5. **CoordinatorDashboardPage** — Read file first. Likely uses SWR for coordinator data.

6. **LandingPage** — Uses `render(<BrowserRouter><LandingPage /></BrowserRouter>)`. This page is entirely synchronous (no SWR, no async content). Add axe test without `await screen.findByRole(...)` — the page renders immediately. Use:
   ```tsx
   it('has no a11y violations', async () => {
     const { container } = render(<BrowserRouter><LandingPage /></BrowserRouter>);
     const results = await axe(container);
     expect(results).toHaveNoViolations();
   });
   ```

7. **LoginPage** — Uses `render(<BrowserRouter><LoginPage /></BrowserRouter>)` with auth-context mock + react-hook-form mocks. Synchronous render (no async data). Add axe test:
   ```tsx
   it('has no a11y violations', async () => {
     const { container } = render(<BrowserRouter><LoginPage /></BrowserRouter>);
     const results = await axe(container);
     expect(results).toHaveNoViolations();
   });
   ```

**General rule:** If the page uses `useSWR`, wait for content: `await screen.findByRole('heading', { name: /.../ })`. If the page renders synchronously (LoginPage, LandingPage, RegisterPage, ResetPasswordPage, etc.), call `axe(container)` immediately after render.

**Violation fix process** (per D-07, same as 17-02):
- Run `npx vitest run src/pages/<Page>.test.tsx -t "has no a11y violations"`
- If violations: identify rule ID + CSS selector → fix production .tsx → re-run → commit
- Carve-out: `// axe-violation: <rule-id> — <reason>` only for unfixable shadcn limitations (per D-09)
- Atomic commit per page: `commit -m "test(a11y): add axe assertion to <Page>"` or `"fix(a11y): resolve <rule-id> in <Page>"`

After all 7 pages, run `npm run test:run` to verify.
