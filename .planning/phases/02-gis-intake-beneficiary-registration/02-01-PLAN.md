    ```typescript
    export const IntakeInputSchema = z.object({
      beneficiary: z.object({
        surname: z.string().min(1),
        firstName: z.string().min(1),
        middleName: z.string().optional(),
        gender: z.enum(['Male', 'Female']),
        dob: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
        barangay: z.string().min(1),
        purok: z.string().optional(),
        phone: z.string().optional(),
        category: z.enum(['Senior', 'PWD', 'Child', 'Solo Parent', 'Indigenous', 'Others']).optional(),
      }),
      familyMembers: z.array(z.object({
        fullName: z.string().min(1),
        relationship: z.string().min(1),
        age: z.number().int().min(0).optional(),
        statusIncome: z.string().optional(),
      })).optional(),
      case: z.object({
        serviceRequested: z.array(z.string()).optional(),
        requirementsChecklist: z.record(z.boolean()).optional(),
        assessedBy: z.string().optional(),
        assignedWorkerId: z.string().optional(),
      }),
    });
    
    export type IntakeInput = z.infer<typeof IntakeInputSchema>;
    ```

    - Commit transaction
    - Return { beneficiaryId, caseId, controlNo, status }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | HTTP POST /api/intake — untrusted input crosses into backend |
| API → Database | TypeORM writes through SERIALIZABLE transaction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | POST /api/intake | mitigate | ZodPipe validates IntakeInputSchema at controller boundary; rejects unknown keys and invalid category enums |
| T-02-02 | Tampering | IntakeService transaction | mitigate | SERIALIZABLE isolation level prevents concurrent control_no collisions; rollbackTransaction() on any failure prevents orphaned records |
| T-02-03 | Spoofing | control_no generation | mitigate | SERIALIZABLE transaction with retry loop (3 attempts, 23505 retry) ensures unique control_no values even under contention |
| T-02-04 | Information Disclosure | Response body | mitigate | Only return beneficiaryId, caseId, controlNo, status — never expose PII in intake response |
| T-02-05 | Tampering | Offline payload | mitigate | Ed25519 signatures on sync (Phase 1); consent rechecked at sync time (not just queue time) |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages — all existing stack (TypeORM, NestJS, Zod, bcrypt) verified in Phase 1 |
</threat_model>
