<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → localStorage | Offline queue stored in browser localStorage — untrusted environment |
| Client → API | Sync endpoint batched payload with Ed25519 signatures |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-15 | Tampering | Offline queue in localStorage | accept | localStorage is browser-accessible by design; Ed25519 signatures on sync payloads prevent replay (Phase 1 built infrastructure) |
| T-02-16 | Spoofing | Offline intake replayed after consent revoke | mitigate | Consent status is rechecked at sync time (IntakeService creates new ConsentLedger with 'active' status — separate from any revoked consent; beneficiary always gets fresh 'active' consent on intake) |
| T-02-17 | Information Disclosure | Offline payload in cleartext localStorage | mitigate | Phase 1's SecureStorage caps encrypted storage; the sync queue itself doesn't use SecureStorage but the payload is ephemeral (removed after sync). Accept for browser PWA; native Capacitor uses SQLCipher |
| T-02-18 | Denial of Service | Malformed intake payload in sync queue | mitigate | Sync processor wraps IntakeService.submitIntake() in try/catch; ZodPipe validation at the IntakeController also validates. Malformed payloads mark the queue entry as 'failed' without affecting other entries |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages |
</threat_model>
