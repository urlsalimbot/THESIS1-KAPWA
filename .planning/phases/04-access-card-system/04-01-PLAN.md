## Phase Goal

**As a** social worker or admin, **I want to** assign a unique Access Card code to a beneficiary in a single step, **so that** every beneficiary has card-based identification and the code is permanently tied to them for reprinting.

<objective>
Refactor the scaffolded card generation from standalone code generation to one-step generate-and-assign per D-01. Add beneficiary card lookup. Wire the client assign flow and reprint-with-identity buttons on the beneficiary detail page.

**Purpose:** Deliver the core AC-01 (card assignment) and AC-03 (reprintable with identity verification) capabilities as a vertical slice: DB → API → UI.
**Output:** atomic generateAndAssign() service method, POST /assign/:id + GET /beneficiary/:id/card endpoints, client assign workflow, beneficiary page buttons.
</objective>
