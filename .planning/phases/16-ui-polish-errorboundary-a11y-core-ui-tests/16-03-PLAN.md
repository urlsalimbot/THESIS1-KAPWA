<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test runtime → Page components | The test mocks `api.get` and/or `useAuth` at the module boundary; the real page subtree is rendered so render-time errors are caught |
| Auth context → Page (role-gated pages) | The test stubs `useAuth` to return a specific role; the page's role-gated UI branches are exercised (e.g., a claimant sees MyAccessCard, an admin sees Programs) |
| SWRConfig → Page (useSWR pages) | The test provides a mock fetcher via `<SWRConfig value={{ fetcher: mockApiGet }}>`; SWR returns the mocked data, the page renders the loaded state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-11 | Tampering | All 8 page test auth mocks | mitigate | Each test mocks `useAuth` to return a specific role. A real attacker would not control this in production. The mock is constrained to the test file by `vi.mock`. The page's real role-gating logic is in the routes config (`routes.tsx`) + the `<ProtectedRoute>` wrapper from 16-02 — both verified independently. |
| T-16-12 | Information Disclosure | Mock api.get responses | accept | The mock returns synthetic data (e.g., `{ id: 'IRF-001', status: 'Under Investigation' }`). No real PII in the test fixtures. |
| T-16-13 | Spoofing | api.get mock returns plausible data | accept | The mock is closed-world (test-only). The test verifies the page renders the loaded state, not the data integrity. The real auth path is Phase 15 scope. |
| T-16-14 | Denial of Service | useParams + raw api.get timing | accept | `useParams` is synchronous; the page's `useEffect` triggers `api.get` which the mock resolves immediately. No real network call. |
| T-16-15 | Elevation of Privilege | Mock user role in role-gated pages | mitigate | The page's role-gated UI is rendered because the mock `useAuth` returns a specific role. The real role enforcement is in `<ProtectedRoute>` (16-02) and the routes config — both verified independently. The test is closed-world. |
| T-16-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed by this plan. |
</threat_model>
