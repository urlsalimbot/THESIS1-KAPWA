<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Page render → ErrorBoundary | Errors from any child component cross here (controlled by react-error-boundary's `onError` callback) |
| ErrorBoundary → EmptyState | The offline branch delegates UI to a public component (no auth, no PII) |
| Test runtime → vitest-axe | The side-effect import in tests/setup.ts is loaded by the vitest test runner only — no production impact |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Tampering | ErrorFallback (offline branch) | accept | An attacker controlling a child could throw `new TypeError('Failed to fetch')` to make the boundary show the offline UI instead of the generic one. This is a deliberate UX feature (ROADMAP #2) — a network failure SHOULD look like an offline failure. No security impact: the offline UI has no destructive actions. |
| T-16-02 | Spoofing | `isOfflineError` predicate | accept | `navigator.onLine` is unreliable in some environments (e.g., captive portals, corporate proxies that intercept 200 responses). The predicate also matches on `error.name === 'TypeError'` + message regex, so it catches fetch failures even when the browser falsely reports online. False positives (offline UI for a non-offline error) are a minor UX papercut, not a security issue. |
| T-16-03 | Information Disclosure | ErrorFallback (offline branch) | accept | The offline UI shows no error details (no stack trace, no error message). The `error` is passed to `isOfflineError` but never rendered. Information disclosure mitigated by the absence of error message in the offline EmptyState. |
| T-16-04 | Denial of Service | `navigator.onLine` check | accept | `typeof navigator !== 'undefined'` guards against SSR/test environments where `navigator` is undefined. The check is synchronous and O(1). No DoS surface. |
| T-16-05 | Elevation of Privilege | EmptyState.onAction | mitigate | The offline branch wires `onAction={resetErrorBoundary}`. `resetErrorBoundary` only re-renders the boundary's children — it does NOT call any auth context, navigation, or storage. The action is constrained to the React render tree. |
| T-16-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed by this plan (vitest-axe is already in devDeps per Phase 15 D-01 verification). The Package Legitimacy Audit in RESEARCH.md verified the existing packages; no new audit required. `T-16-SC` in the threat model is informational. |
</threat_model>
