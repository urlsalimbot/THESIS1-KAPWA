<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client→API | Preference update requests cross this boundary (untrusted input) |
| API→Twilio | SMS message delivery (relies on Twilio for actual delivery) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | PUT /notifications/preferences | mitigate | JwtAuthGuard + @Roles ensures user can only set own preferences (recipientId derived from JWT, not body) |
| T-06-02 | Tampering | sendWithConsent | mitigate | Consent check occurs server-side in service layer (D-06), not client-side — controller delegates to service method |
| T-06-03 | Information Disclosure | GET /notifications/preferences | mitigate | Returns only req.user.id's preferences — no parameter for querying others |
</threat_model>
