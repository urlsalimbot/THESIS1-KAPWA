## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Page component ↔ useSWR (global fetcher) | The page passes a key tuple to useSWR; SWR calls api.get (the global fetcher set in routes.tsx); the response is JSON parsed by the api client |
| Page component ↔ useSWRMutation | The page passes a key and a fetcher function; on trigger, useSWRMutation calls the fetcher which uses api.put / api.post / api.del; the response becomes the mutation data |
| Page component ↔ local React state | useState for UI state (search input, active tab, modals, form fields); NOT for server data (server data flows through useSWR) |
| auth-context.tsx ↔ api.ts | auth-context.fetchUser calls api.get('/auth/me'); the api client reads the token from localStorage internally (D-02) |
| FilingPage handleUpload/handleDownload ↔ raw fetch | FormData upload and blob download are deliberate carve-outs (D-10 deferred) — they bypass the api client and use raw fetch with manual Bearer header |
| Test code ↔ vi.mock('../lib/api') | The test mock provides canned responses for api.get / api.put / api.post / api.del; the real api client is bypassed |
